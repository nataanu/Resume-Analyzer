Overview

Why the Colab Edition?

Google Colab runs on a headless Ubuntu VM — there is no graphical display server ($DISPLAY variable is unset). Any attempt to open a Tkinter window fails immediately with:

TclError: no display name and no $DISPLAY environment variable

This edition solves the problem entirely by replacing Tkinter with ipywidgets — UI components that render as HTML inside the Colab notebook output cell, with no OS-level window required.

🌐
Browser-native

Usage

How to Use

  • 1
    Run the setup cell — Install libraries and download NLTK data.
  • 2
    Run the script cell — The widget UI appears inline below the cell output.
  • 3
    Browse Resume — Click the file upload button and select your .pdf or .docx resume. File size and name appear in green on success.
  • 4
    Paste JD — Click inside the Job Description textarea and paste the full job description.
  • 5
    Analyze — Click "Analyze Resume". The status bar turns amber while processing, then green with your score.
  • 6
    Read the Report — A fully styled HTML report appears inline with an SVG score ring, skill pills, and suggestions.
  • 7
    Clear — Click the Clear button to reset all fields and run a new analysis.

Architecture

Module Breakdown

#ModuleResponsibility
1Text ExtractionReads bytes from browser upload via io.BytesIO — pdfplumber → PyPDF2 fallback for PDF; python-docx for DOCX
2PreprocessingLowercase → compound protection → punctuation removal → tokenise → stopwords → lemmatise
3Skill ExtractionWord-boundary regex matching against 100+ taxonomy skills across 10 domains
4Scoring EngineWeighted composite: Skill Match (70%) + Keyword Overlap (30%)
5SuggestionsBand-based recommendations + missing skill pills + quantification & structure tips
6Pipelineanalyze_resume(filename, file_bytes, jd) — orchestrates modules 1–5 end-to-end
7HTML Report BuilderBuilds self-contained HTML with inline SVG score ring, skill pills, dark-themed stats grid
8Colab UIlaunch_ats_app() — renders all ipywidgets and handles on_click callbacks
Scoring

ATS Score Formula

# Skill Match Score (70%)
skill_score = len(resume_skills ∩ jd_skills) / len(jd_skills) × 100

# Keyword Overlap Score (30%)
kw_score    = len(resume_tokens ∩ jd_tokens)  / len(jd_tokens)  × 100

# Composite ATS Score
ats_score   = round(0.70 × skill_score + 0.30 × kw_score, 1)

Limitations

Known Limitations

  • Scanned / image-only PDFs — text extraction will fail (OCR not included)
  • Niche skills not in SKILL_TAXONOMY will be missed
  • No semantic matching — 'ML' and 'machine learning' treated as different tokens
  • English-only — stopwords and lemmatisation are English only
  • No persistence — results not saved between sessions; no database
  • No downloadable export — HTML report is inline only (no PDF save)
  • Requires cell re-run after Colab session restart to reload the widget
