Overview

What is this?

The ATS Resume Analyzer — VS Code / Local Edition is a full-featured desktop application built with Python's Tkinter framework. It runs as a native OS window on Windows, macOS, or Linux and requires an active graphical display (X11/Wayland).

Upload your resume (PDF or DOCX), paste a job description, and receive an instant composite ATS compatibility score — broken down into skill match and keyword overlap components — along with a tabbed report showing matched skills, missing skills, improvement suggestions, and full stats.

🖥️
Native Desktop

🎯

pip

Installation

Quick Start

Step 1 — Install dependencies

Terminal / Command Prompt
pip install pdfplumber python-docx nltk

Step 2 — Download NLTK data (run once)

Python shell or script
import nltk
nltk.download('stopwords')
nltk.download('wordnet')
nltk.download('punkt')

Step 3 — Launch the application

Terminal
python ats_resume_analyzer_vs.py
⚠️
Windows users: If you installed Python from the Microsoft Store, Tkinter may be missing. Reinstall Python from python.org with the "tcl/tk and IDLE" option checked.
🐧
Linux headless servers: Tkinter requires a display. Use export DISPLAY=:0 or run the Google Colab edition instead.
Usage

How to Use

  • 1
    Launch — Run the script. A dark-themed window (1160×820) opens on your desktop.
  • 2
    Browse Resume — Click "📂 Browse PDF / DOCX" and select your resume file. Supported: .pdf, .docx, .doc.
  • 3
    Paste JD — Click inside the Job Description area and paste the full job description text.
  • 4
    Analyze — Click "🔍 Analyze Resume". The progress bar animates while the analysis runs in the background.
  • 5
    Review Results — Four tabs appear: Matched Skills, Missing Skills, Suggestions, and Full Report.
  • 6
    Reset — Click Reset to clear everything and start a fresh analysis without restarting the app.
Architecture

Module Breakdown

#ModuleResponsibility
1Text ExtractionReads raw text from PDF (pdfplumber → PyPDF2 fallback) or DOCX via direct disk filepath
2PreprocessingLowercase → compound protection → punctuation removal → tokenise → stopwords → lemmatise
3Skill ExtractionWord-boundary regex matching against 100+ taxonomy skills across 10 domains
4Scoring EngineWeighted composite: Skill Match (70%) + Keyword Overlap (30%)
5SuggestionsBand-based recommendations + missing skills + quantification & structure tips
6Pipelineanalyze_resume(filepath, jd) — orchestrates modules 1–5 end-to-end
7Tkinter GUIScoreRing canvas widget, Tooltip system, ATSApp with tabbed notebook, threading, status bar
Scoring

ATS Score Formula

# Skill Match Score (70%)
skill_score = len(resume_skills ∩ jd_skills) / len(jd_skills) × 100

# Keyword Overlap Score (30%)
kw_score    = len(resume_tokens ∩ jd_tokens)  / len(jd_tokens)  × 100

# Composite ATS Score
ats_score   = round(0.70 × skill_score + 0.30 × kw_score, 1)
🏆 Excellent
≥ 80%
👍 Good
60–79%
🤔 Moderate
40–59%
⚠ Low
< 40%

Limitations

Known Limitations

  • Scanned / image-only PDFs — text extraction will fail (OCR not included)
  • Niche or domain-specific skills not in the taxonomy will be missed
  • No semantic matching — 'ML' and 'machine learning' are treated as different unless both appear
  • English-only — stopwords and lemmatisation are English only
  • No persistence — results are not saved between sessions
  • No export — cannot save the report as a file from within the app
